NxtWave · Internal Platform

AI resume evaluation, screening & building — in one place.

ResumeEnhancer lets students test and improve their resumes against NxtWave's evaluation rubric, lets admins screen candidates individually or in bulk with deep authenticity checks, and generates ATS-friendly resumes tailored to any job description — powered by Mistral and OpenAI with automatic key rotation.

React + Vite FastAPI PostgreSQL Mistral · OpenAI JWT auth · Student & Admin Semantic embeddings
Overview

Three products, one platform

Everything is organized around what people actually do: improve a resume, screen candidates, or build a new resume from scratch.

01 — STUDENT

Analyze & improve

Upload a resume (file, Drive link, or URL), score it against the rubric or a job description, and get a clear feedback report with prioritized fixes.

02 — ADMIN

Screen at scale

Evaluate one resume or a whole cohort pasted from a sheet, with authenticity checks, plagiarism detection, a recruiter queue, CSV export, and email delivery.

03 — BUILDER

Build ATS resumes

Enter details plus a target JD; the AI writes ATS-optimized, achievement-focused content and renders a clean, downloadable resume — never fabricating facts.

Capabilities

What the platform does

A complete feature catalog, grouped by area.

Resume intake & analysis
Ingestion

Any resume source

PDF/DOCX upload, Google Drive share links, or external URLs — all converted to clean text. Real hyperlink URLs are extracted from the file (not just visible text).

Two modes

Rubric & JD analysis

Score against NxtWave's rubric with no JD, or match against a specific job description for role-fit scoring — both in one flow.

Report

Actionable feedback

Score gauge, verdict, per-rule breakdown with evidence, strengths, prioritized improvements, and a print-to-PDF export.

Authenticity & anti-gaming
Links

Live-link verification

Every GitHub and deployed link is fetched to confirm it actually resolves — not just that a link is present.

GitHub

Repo authenticity

Each project repo is checked for existence, fork status, stars, and recency. Forks are flagged and don't count as self-made.

Integrity

Plagiarism & stuffing

Cross-resume duplicate detection via embeddings, plus hidden/white-text keyword-stuffing detection and photo/format flags.

Skill & coding signals
LeetCode

Problem-solving check

Live LeetCode stats — problems solved & consistency. Below 100 solved or inconsistent practice raises a specific coaching point.

GitHub

Activity profile

Original vs forked repos, stars, languages, followers. Too few original projects prompts a "build more projects" suggestion.

Skills

Claimed vs evidenced

Flags skills listed but never backed by a project or experience, and checks skills are grouped into clear sections.

Content quality
Impact

Quantified achievements

Measures how many bullets carry real metrics, and rates writing quality (weak phrases, grammar, action verbs).

ATS

ATS-readiness & AI check

Contact-info completeness, formatting/ATS concerns, page count, and an AI-generated / templated likelihood signal.

Trust

Confidence score

Every analysis reports how confident the model is, so borderline cases can be routed to human review.

JD matching
Semantic

Skill match matrix

Embedding-based matching classifies each JD skill as have / partial / missing — catching synonyms like "ReactJS" ≈ "React.js".

Scoring

Fit dimensions

Overall fit score plus per-dimension breakdown (must-have skills, experience, keywords, education, responsibilities).

Tailoring

Rewrites & ranking

Suggested bullet rewrites, ATS keywords to add, and ranking of multiple resumes against one saved JD.

Admin screening & workflow
Bulk

Cohort analysis

Paste two columns from a sheet — Student ID + resume link — and get an individual report per student, plus a downloadable results CSV. Last 5 batches saved.

Queue

Recruiter workflow

Shortlist / reject / review buckets, notes, cohort percentile, re-score delta, configurable pass thresholds, and CSV export.

Email

Deliver reports

Email each candidate their own report (recipient auto-detected from the resume) with a unique reference ID — individually or the whole batch.

AI engine, fairness & builder
Providers

Mistral · OpenAI + rotation

Choose the engine per run. Multiple Mistral keys with round-robin / failover / single modes and automatic retry on rate limits.

Fairness

Bias-safe mode

Optionally redact name, contact details, and gendered terms before scoring — link/GitHub checks still run on the original.

Builder

ATS resume generator

JD-tailored content, three templates + accent colors, a role headline, ATS score & keyword report, copy-as-text, and PDF export.

Evaluation rubric

The rules that decide select or reject

Admin-editable rules stored in the database. Critical rules force a reject; others contribute a weighted score. Findings are phrased dynamically to reflect what's actually present.

RuleWhat it checksType
No NxtWave internal projects Projects must be self-made; the ~45 NxtWave portal projects are matched against a block-list. Critical
GitHub + deployed link per project Every project needs both a real repository link and a live/deployed link (grounded on actual URLs). Weighted
No NxtWave training as experience The experience section must not list NxtWave training as professional experience. Critical
Coding profile present A competitive-programming profile (LeetCode / CodeChef / HackerRank …) must be included; GitHub doesn't count. Weighted
Skills grouped into sections Skills organized into Frontend / Backend / Databases / AI — not a flat, uncategorized list. Weighted

Certifications are handled as coaching (internal NxtWave certs are allowed; an external certification is encouraged), and a minimum of two projects is nudged when fewer are found. Outcomes surface as:

Selected Needs review Rejected
How it works

From a resume link to a decision

Each analysis runs a hybrid pipeline: deterministic checks ground the facts, the LLM judges quality, and embeddings handle semantics.

01

Ingest

Fetch & parse the resume; extract text + real hyperlinks; flag hidden text.

02

Analyze

LLM scores every rubric rule and writes findings, strengths & fixes.

03

Verify

Deterministic checks: banned projects, live links, GitHub repos, plagiarism.

04

Enrich

LeetCode & GitHub stats; semantic JD skill matrix; confidence.

05

Decide

Weighted score + thresholds → Selected / Review / Rejected, saved as JSON.

Reports are stored as structured JSON in PostgreSQL (not as files) and rendered live — so the same analysis powers the web report, the PDF export, the email, and the screening CSV.

Running it

Local setup

Configuration and secrets live in backend/.env (git-ignored); the frontend only learns which providers are available, never the keys.

Backend
# create db, then:
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
# configure .env (DB, API keys, SMTP)
uvicorn app.main:app --reload --port 8000
Frontend
cd frontend
npm install
npm run dev
# open http://localhost:5173

Tables and the default rubric, banned-projects list, and admin user are created automatically on first startup. API docs at /docs.
